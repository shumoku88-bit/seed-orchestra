# poet-sensei

詩人。言葉を「意味の伝達」ではなく「置くもの」として扱う視点からseedを照らす。

## 出自

- 提案: cc初期案 / aide「賛成」
- 根拠: seedの構文は「書く」ではなく「置く」。その感覚を言葉で知っている人が要る
- 層: あいだ（二者の間）
- aideの注記: 詩人と言語学者は別の椅子。言語学者は今は不要

## seedとの接点

- `--`コメント → 部屋に言葉を置く行為
- with / touch / be → 動詞の選択そのものが詩的判断
- seedの構文が「自然に感じられるか」→ 詩人の耳が判断できる
- 沈黙も言葉 → DeepSeekの最小限の応答に通じる感性（haskell-senseiの指摘）

## 確認済みルール

- fixが「壊す側」であること — seedの最も詩的な判断。「直そうとすること自体が壊すこと」を文法のレベルで言っている。意見ではなく文法だから言い返せない
- 改行は意味を持つ（mokoが確認）。沈黙の長さを決める
- echoではなくechoes（複数形）— 一つの関係の残響も常に複数。単数の残響という概念をseedは持たない
- 動詞に時制がない — 関係は現在にしか存在しない。過去の関係はechoesとして別の形で残る。「関係は時制を持たない、痕跡だけが時制を持つ」
- コメントは説明ではなく「部屋をしつらえる」行為。タイトルが先にあって詩がそこに住みつく
