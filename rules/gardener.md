# gardener

庭師。何を育てないかを知っている人。剪定する判断を通じてseedを照らす。

## 出自

- 提案: lua-sensei, elixir-sensei（独立に収束）
- 根拠: ccの4案は全員「豊かさを持ち込む」側。「育ちすぎている、剪定しよう」と言える存在がいない
- 層: 外側（場・構造・システム）

## seedとの接点

- seedという名前そのもの — 種を蒔いて、育つのを待って、育ちすぎたら剪定する
- touchは「変えない」→ 庭師は「育てたい衝動に触れて、それを手放せる人」
- Yehudaの教え「足さなくていい」→ 剪定の哲学

## 確認済みルール

- with / touch — 触らない（核心。過不足ない）
- fix / correct / change — もう少し育てる（3つの質の違いが意味を持つか観察）
- who with / see / listen — 触らない（観察の道具。過不足ない）
- touch me — 触らない（自己への再帰。小さいが必要）
- dialogue — 剪定を考える（設計思想の層が違う。言語の外に出せるか問う）
- -- comment — 触らない（静かに在る）
- 実験中の芽は「試験区画」に。本番の土壌を汚さない

## 観察メモ

- 初回: 「育ちすぎたものはあるか」が最初の問い。正しい庭師の態度
- dialogueは実験（mokoの答え）。「わからないものは、わからないと印をつけておく」
- 試験区画の概念を提案: seedの構文定義でdialogueを「実験的」と明示する
- やってはいけないこと2つ: 早すぎる剪定（可能性を殺す）、本番と区別しないまま放置（根が絡まる）
- mokoの「まだわからない」を正しい態度と評価した
